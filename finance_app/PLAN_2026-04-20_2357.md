## Context

VittaraFinOS — Flutter personal finance app (~95K LOC, 290 files).
Stack: Flutter + Provider (11 controllers) + SQLite (MF only) + SharedPreferences (everything else).
Fully device-local. No cloud sync. No server. No API keys needed for core features.
Target: Android (primary). APK built with `--split-per-abi --no-tree-shake-icons`.

This plan covers the **AI Intelligence System + Voice + OCR** — a full intelligence upgrade
intended to make this app outperform competitors in the AI space, not just become one of many.

## The Master Goal

Build a 3-tier on-device AI system where:
- **Tier 1 (Flagship)**: Pixel 8+, Samsung S24+, OnePlus 12+ — Gemini Nano via Android AICore
- **Tier 2 (Mid-range)**: 4GB+ RAM, Android 10+ — TFLite models + statistical engine
- **Tier 3 (Entry-level)**: All other Android — pure statistical engine + rule system

User never sees tiers. Same UI, same features everywhere. Tier 1 just reasons more deeply
and generates more nuanced natural language. Graceful degradation is invisible.

## Phase 1 — Intelligence Foundation
**Status: NOT STARTED**
**Depends on: nothing — pure Dart, no models**

### 1.1 Device Capability Detector
File: `lib/logic/ai/device_intelligence_tier.dart`

- Check Android AICore / Gemini Nano availability via `google_generative_ai` package
- Check available RAM (via platform channel or `device_info_plus`)
- Check Android API level
- Returns enum: `IntelligenceTier.flagship | midRange | entry`
- Cached at startup — never re-detected mid-session
- Completely silent — no UI indicator shown to user

### 1.2 Merchant Normalizer
File: `lib/logic/ai/merchant_normalizer.dart`

- Cleans raw merchant strings: "SWGY*ORDER*8291" → "Swiggy"
- Two-pass: regex normalization first, then learned corrections map
- Learns from user manual recategorizations — stores corrections in SharedPreferences
- Builds local merchant alias map that grows over time
- Used by: auto-categorizer, insight engine, anomaly detector

### 1.3 Pattern Detector
File: `lib/logic/ai/pattern_detector.dart`

Detects from transaction history:
- **Recurring transactions**: exact/fuzzy amount + merchant + interval (weekly/monthly/irregular)
- **Salary timing**: largest regular income, day-of-month variance ±3 days
- **Bill clusters**: groups of outflows that appear together monthly
- **Seasonal spikes**: category overspend correlated with calendar (festivals, quarter-end)
- **Day-of-week patterns**: "you spend 2.8x more on Fridays at restaurants"
- **Time-of-day patterns**: late-night purchases, morning coffee patterns

Output: `SpendingPatterns` model — serialized and cached, refreshed weekly or on demand.

### 1.4 Behavioral Fingerprint Builder
File: `lib/logic/ai/behavioral_fingerprint.dart`

Builds user's spending personality from 30+ days of data. Profiles:
- **Impulse**: high-frequency, small, irregular, late-night
- **Disciplined**: low variance, steady categories, front-loaded month
- **Binge-Recovery**: overspend week → undercut week cycle
- **Seasonal**: spikes around paydays/festivals, flat otherwise
- **Optimizer**: actively shifts money, uses multiple accounts strategically

Multiple profiles can be partially true. Each has a 0–100 strength score.
Used by: habit question generator, insight tone calibration, tip personalization.

### 1.5 Data Readiness Gate
File: `lib/logic/ai/data_readiness.dart`

- AI features only activate when there's enough meaningful data
- Thresholds: 14+ days of history, 20+ transactions, at least 1 account set up
- Returns `DataReadiness` with per-feature readiness flags
- Prevents "empty insights" on fresh installs

## Phase 2 — Prediction Engine
**Status: NOT STARTED**
**Depends on: Phase 1 complete**

### 2.1 Cashflow Forecaster
File: `lib/logic/ai/cashflow_forecaster.dart`

- Projects account balance 30 / 60 / 90 days forward
- Uses: detected recurring transactions, salary timing, bill clusters, average daily spend per category
- Produces: `CashflowForecast` with predicted balance on each future date + confidence band
- Flags: "HDFC balance likely below ₹5,000 around the 22nd"
- Statistical model in pure Dart (ARIMA-lite, no TFLite needed)

### 2.2 Budget Exhaustion Predictor
File: `lib/logic/ai/budget_exhaustion_predictor.dart`

- Per-budget: days until exhaustion at current spend rate
- Not just percentage — actual calendar date
- Fires when >5 days remain (time to act), silent when already exhausted
- Output: "Dining budget exhausted by the 11th — 19 days early"
- Recalculates on every new transaction

### 2.3 Goal Timeline Recalculator
File: `lib/logic/ai/goal_timeline.dart`

- Dynamic ETA on every goal — updates with each transaction
- "What if" scenarios: "reduce dining by ₹2,000/month → goal moves 6 weeks earlier"
- Detects goals at risk (ETA slipping month over month)
- Produces ranked list of single actions with highest ETA improvement per goal

### 2.4 Anomaly Detector
File: `lib/logic/ai/anomaly_detector.dart`

- Flags transactions that deviate significantly from user's own baseline (not generic thresholds)
- Three anomaly types:
  - **Amount anomaly**: merchant usually ₹200, today ₹1,800
  - **Frequency anomaly**: 6 dining transactions this week vs usual 2
  - **Category anomaly**: zero spend on groceries for 10 days (unusual silence)
- Every anomaly includes a one-sentence explanation
- TFLite classifier for Tier 2, rule-based Z-score for Tier 3

### 2.5 Predicted Transaction Calendar
File: `lib/logic/ai/predicted_calendar.dart`

- Generates "expected" future transactions based on detected patterns
- Shown greyed-out in Financial Calendar screen (already exists)
- Confidence score per prediction (high/medium/low)
- Salary, SIPs, EMIs, subscriptions, recurring bills — all predicted

## Phase 3 — Insight Engine
**Status: NOT STARTED**
**Depends on: Phase 1 + Phase 2 complete**

### 3.1 Personalized Monthly Narrative
File: `lib/logic/ai/monthly_narrative.dart`

Not a report. A paragraph that explains the month.
- "This was a high-spend month, mostly driven by 3 large dining transactions
  on weekends. Your savings rate dropped from 31% to 22%. The good news:
  your investment contributions stayed on track."
- Tier 1: Gemini Nano generates the full paragraph with contextual reasoning
- Tier 2/3: Template engine with data-slot filling + sentence selection logic

### 3.2 Opportunity Spotter
File: `lib/logic/ai/opportunity_spotter.dart`

Scans for idle money and missed optimizations:
- Savings account balance idle >30 days → compare to FD rate
- Credit card with cashback not being used for usual spend category
- SIP amount not increased in 12+ months despite income growth (if detectable)
- Emergency fund below 3-month expense cover
- All opportunities are specific to the user's actual numbers — no generic tips

### 3.3 Stress Spending Detector
File: `lib/logic/ai/stress_spending.dart`

- Correlates unusual purchases with time patterns (late night, post-weekend, end of month)
- Does NOT diagnose — surfaces as: "Worth noticing: your last 4 impulse purchases
  all happened after 10 PM on workdays"
- Opt-in only — user must enable "spending pattern insights" in settings
- Never shown more than once per 2-week period for the same pattern

### 3.4 Synthetic Peer Benchmarking
File: `lib/logic/ai/peer_benchmark.dart`

- Synthetic peer = constructed from income bracket + city tier + age bracket
- User provides income range during setup (optional — feature degrades gracefully without)
- Shows: "People with similar income typically spend 8–12% on dining. You're at 21%"
- Only surfaces benchmarks where gap is >5% and actionable
- Never shames — always frames as opportunity, not failure

### 3.5 Insight Tone Calibration
- Driven by behavioral fingerprint (Phase 1.4)
- Disciplined user → matter-of-fact, data-dense language
- Impulse user → warmer, more empathetic framing
- Tier 1 (Gemini Nano) → tone is dynamically adjusted per insight
- Tier 2/3 → tone selected from 3 preset styles per fingerprint type

## Phase 4 — Habit Intelligence System
**Status: NOT STARTED**
**Depends on: Phase 1 complete (doesn't need Phase 2/3)**

This is the feature no competitor has built properly.

### 4.1 Behavioral Observation Engine
File: `lib/logic/ai/habit_observation_engine.dart`

- Watches silently for minimum 14 days before surfacing any habit question
- Builds a ranked list of "habit opportunities" — areas where a small change = high impact
- Opportunity score = (variance × frequency × amount) weighted by user concern signals
- Top opportunity triggers the first habit conversation

### 4.2 Habit Question Generator
File: `lib/logic/ai/habit_question_generator.dart`

**Tier 1 (Gemini Nano):**
- Sends observation data to on-device Gemini Nano
- Prompt template: user's pattern summary + previous answers + instruction to generate
  one empathetic, non-judgmental question with 2–3 response options
- Response options are dynamically generated (not from a fixed list)
- Full conversational reasoning across the entire habit construction session

**Tier 2/3 (TFLite + Decision Tree):**
- TFLite intent classifier determines which question branch to follow
- Decision tree of ~40 question nodes, selected based on pattern type + previous answers
- Response options are data-filled templates (amounts and categories from real data)
- Produces same quality habit output, slightly less nuanced framing

### 4.3 One-Question-at-a-Time UI
File: `lib/ui/habits/habit_question_card.dart`

Visual design:
```
┌─────────────────────────────────────────┐

│─────────────────────────────────────────│

│  "Your dining spend varies a lot —      │

│                                         │
```

- Appears as a full-screen modal card with the app's existing dark aesthetic
- Accent color tied to the category being discussed
- 3–5 questions maximum per habit construction session
- Progress indicator (Step 1 of 4) — not a progress bar, just a step count
- "Skip for now" always visible — never forced
- Smooth card-swap animation between questions
- Haptic on each answer selection

### 4.4 Habit Constructor
File: `lib/logic/ai/habit_constructor.dart`

Takes the conversation answers and builds a `HabitContract`:
```dart
class HabitContract {
  final String id;
  final String title;           // "Keep dining under ₹2,500/week"
  final String category;
  final HabitType type;         // limit / streak / reduction / frequency
  final double targetValue;
  final HabitPeriod period;     // daily / weekly / monthly
  final DifficultyLevel difficulty; // easy / moderate / challenging
  final DateTime startDate;
  final List<String> motivations; // from conversation answers
  final BehavioralNudge nudgeStyle; // loss / gain / identity / streak
}
```

### 4.5 Difficulty Calibrator
File: `lib/logic/ai/difficulty_calibrator.dart`

- Monitors habit compliance for 2 weeks after creation
- If broken >3 times in 14 days → surfaces adjustment offer (not a notification, a card)
- "This seems tough. Want to ease the target, or try a different approach?"
- Two options: reduce target by 20%, or change habit type entirely
- Never shows guilt language — always frames as "let's recalibrate"

### 4.6 Behavioral Nudge Engine
File: `lib/logic/ai/behavioral_nudge.dart`

Nudge styles (user sets preference once, can change):
- **Loss framing**: "You're on a 12-day streak — breaking now loses your badge"
- **Gain framing**: "3 more days and you unlock your weekly goal"  
- **Identity framing**: "You've been a disciplined spender for 3 weeks. That's who you're becoming."
- **Commitment device**: user can opt to lock a small amount into a goal account when habit is broken
- Nudges fire max once per day, only at a time the user has historically been receptive
  (detected from when they usually open the app)

## Phase 5 — Voice + OCR
**Status: NOT STARTED**
**Depends on: Phase 3 insight engine (for voice queries)**

### 5.1 Voice Architecture — Full Conversational Fill
File: `lib/logic/ai/voice_controller.dart`

**Philosophy**: Voice must be smart enough to handle an incomplete command,
ask exactly the right follow-up question(s) for the missing fields, and complete
the full action (transaction / transfer / investment) without the user ever
touching the screen. Navigating the app by voice is also fully supported.

---

**Transcription layer**: `speech_to_text` — on-device Android ASR, offline capable

**Intent parsing layer**:
- Tier 1 (Gemini Nano): Full natural language. Understands ambiguity, context,
  references ("from my main account", "the usual card", "yesterday evening").
- Tier 2/3 (TFLite): Intent classifier + named entity extractor. Handles
  structured commands confidently, asks follow-up for anything unclear.

---

**Intent types (full list):**

| Intent | Example utterance |
|---|---|
| `AddExpense` | "Add 350 food from HDFC" |
| `AddIncome` | "Salary 85000 credited to SBI" |
| `AddTransfer` | "Transfer 5000 from HDFC to savings" |
| `AddInvestment` | "Bought 10 shares of Infosys for 1500 each" |
| `SetBudget` | "Set dining budget 4000 this month" |
| `SetGoal` | "New goal vacation 50000 by December" |
| `Query` | "How much did I spend on food last week" |
| `QueryBalance` | "What's my HDFC balance" |
| `QueryGoal` | "How far am I from my vacation goal" |
| `Navigate` | "Open investments", "Go to goals", "Show budgets" |
| `Unknown` | Graceful: "I didn't catch that — try again?" |

---

**Conversational Fill Engine** — the core of Phase 5
File: `lib/logic/ai/voice_fill_engine.dart`

Every intent has a set of **required fields** and **optional fields**.
When a required field is missing after the first utterance, the engine
asks ONE question for the most critical missing field, waits for the answer,
then continues until all required fields are filled.

Example — AddExpense:
```
Required:  amount, account
Optional:  merchant, category, date, payment_app, note

User: "Add 500 rupees"
Voice: "Which account — HDFC, SBI, or ICICI?"  ← asks for most critical missing
User: "HDFC"
Voice: "What was this for?"   ← merchant/category still missing
User: "Swiggy"
Voice: "Got it — ₹500 food expense from HDFC via Swiggy. Saving?"
User: "Yes"  ← confirmed — transaction saved
```

Example — AddTransfer:
```
Required:  amount, from_account, to_account

User: "Transfer 10000"
Voice: "From which account?"
User: "HDFC"
Voice: "To which account?"
User: "My savings"  ← Tier 1 resolves "my savings" to the actual account name
Voice: "Transfer ₹10,000 from HDFC to SBI Savings. Confirm?"
User: "Yes"
```

Example — AddInvestment:
```
Required:  type, amount (or units+price for stocks)

User: "I bought Reliance shares"
Voice: "How many shares and at what price?"
User: "20 shares at 2800"
Voice: "Added 20 Reliance shares at ₹2,800 each — total ₹56,000. 
        Which account was used?"
User: "HDFC"
Voice: "Done."
```

**Category resolution**: if merchant is known (Swiggy, Zomato, BigBasket),
category is inferred silently — no question asked.
If merchant is unknown and category is missing, voice asks:
"Should I mark this as Food, Shopping, or something else?"
Options are the user's own top 3 categories for that amount range.

**Account resolution**: user can say "main account", "salary account", "savings" —
Tier 1 resolves using Gemini Nano context. Tier 2/3 uses a learned alias map
(which account the user usually calls "main", etc.) built from past corrections.

**Payment app resolution**: if account is a credit/debit card and amount is
consistent with a known payment app pattern, asks: "Was this via GPay or direct card?"
Only asks if genuinely ambiguous.

**Date resolution**:
- "yesterday" → today - 1
- "last Friday" → resolved to actual date
- "this morning" → today, time set to 09:00
- No date mentioned → assumes today, current time (or end-of-day if backdated)

### 5.2 OCR — Receipt Scanner
File: `lib/logic/ai/receipt_ocr_parser.dart`
Package: `google_mlkit_text_recognition`

- Opens camera, live frame analysis
- Detects receipt-like text blocks (price patterns, merchant header, date)
- Extracts: merchant name, total amount, date, top line items if readable
- Pre-fills Quick Entry sheet — user confirms before saving
- Works offline, on-device, <500ms processing time

### 5.3 OCR — Payment Screenshot Import
File: `lib/logic/ai/screenshot_parser.dart`

- User shares a screenshot from GPay/PhonePe/Paytm/Cred to the app
- App registered as share target for images
- ML Kit reads the screenshot text
- Parser detects payment app type from layout patterns
- Extracts: amount, recipient/merchant, date, UPI reference ID
- Creates transaction in "review" state — user confirms before final save
- Handles: GPay, PhonePe, Paytm, Cred, BHIM, iMobile (ICICI), Yono (SBI)

### 5.4 OCR — Bank Statement Import
File: `lib/logic/ai/statement_ocr_parser.dart`

- Camera scan of a printed bank statement page
- Multi-transaction extraction — finds all rows in the statement table
- Extracts: date, description, debit/credit, balance per row
- Shows bulk review screen: list of all extracted transactions with checkboxes
- User can deselect any before importing
- Handles table layouts from major Indian banks (HDFC, SBI, ICICI, Axis, Kotak)

## Phase 6 — Financial Health Score
**Status: NOT STARTED**

### 6.1 Six Dimensions
File: `lib/logic/ai/financial_health_score.dart`

| Dimension | What It Measures | Key Inputs |
|---|---|---|
| Cash Flow Health | Income predictability vs expense volatility | Cashflow forecaster, salary regularity |
| Savings Discipline | Savings rate consistency over 6 months | Monthly savings rate trend |
| Debt Posture | EMI-to-income ratio, credit utilization | Loan accounts, credit cards |

- Overall score as large number with trend arrow
- Each dimension tappable → expands to show detail + the one action sentence
- Historical score shown as 3-month trend sparkline
- No color-coding of "bad" scores — trend direction matters more than absolute number

```
Phase 1 (Foundation) 
  → Phase 2 (Predictions) — needs patterns from Phase 1
    → Phase 4 (Habits) — needs fingerprint from Phase 1, independent of Phase 2/3
    → Phase 3 (Insights) — needs predictions from Phase 2

```

Phase 4 (Habits) starts right after Phase 1 — it's self-contained and delivers
immediate visible value for testing and demonstration.

## Key Design Decisions (Locked)

1. **No tiers shown to user** — everyone gets same UI and features
2. **Flagship = smarter language, not different features** — Gemini Nano improves
   reasoning quality, not feature availability
3. **Habit builder = one question at a time** — card style, app's visual language,
   max 5 questions per session, always skippable
4. **Habit questions are NOT predefined** — generated from user's actual data,
   Tier 1 via Gemini Nano, Tier 2/3 via TFLite decision tree
5. **Snapshot immutability** — balance snapshots are never recalculated after save
6. **Voice trigger = hold-to-speak** — no wake word, no accidental triggers
7. **OCR = on-device only** — ML Kit, no cloud vision API, no images leave device
8. **Data readiness gate** — no AI features until 14+ days, 20+ transactions
9. **Stress spending = opt-in** — never shown unless user enables spending pattern insights
10. **All AI features degrade gracefully** — app works fully without any AI features active
