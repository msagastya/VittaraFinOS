## VittaraFinOS — Full App Responsive Design Master Plan
### Every Screen · Every Device · 14 Sprints

---

## Context & Problem Statement

The app was designed and visually tuned on **Samsung Galaxy S24 Ultra (~480dp logical width,
flat display)**. On **Motorola G-series (~393dp logical width)** and other narrower phones:

- Dashboard cards show large empty whitespace at the bottom
- Content does not fill the card height — it shrink-wraps
- Everything looks proportionally smaller
- The same issue cascades to every screen in the app

**Root cause (confirmed by audit):**
1. `_buildHeaderSection` in `main.dart` has no height cap — on phones with identical height but
   less width it consumes an unfair vertical share, starving the CardDeck below it
2. `NetWorthWidget.buildContent()` wraps content in its own `SingleChildScrollView` — but the
   outer `_buildDashboardWidgetCard` in `main.dart` also wraps in `SingleChildScrollView` with
   `ConstrainedBox(minHeight: constraints.maxHeight)`. The inner scroll shrink-wraps, defeating
   the outer minHeight constraint
3. 2,126 hardcoded `fontSize:` values across 63 files — not scaling with device width
4. 193 structural `height: N` values above 40dp — not scaling with device dimensions
5. 16 files with fixed `childAspectRatio` — breaks on narrow phones
6. 67 files with bottom sheets/modals — none constrained for tablet/fold inner screens
7. `ResponsiveTypography` class already exists in `typography.dart` — but is dead code, never
   called anywhere in the UI

## Rules for Every Sprint

1. **Never modify `Spacing.*` or `TypeScale.*` constants** — they are `const` used at 500+ sites.
   Add responsive equivalents as new classes/methods only.
2. **Never override `MediaQuery.textScaler` globally** — already clamped to [0.85, 1.3] in
   `main.dart`. Further overrides break CupertinoNavigationBar and accessibility.
3. **Use `LayoutBuilder` instead of `MediaQuery.of(context).size`** inside widget trees —
   `LayoutBuilder` gives the actual constrained size, not the screen size.
4. **Never use `Expanded` inside a `SingleChildScrollView`** — causes unbounded height errors.
5. **Test on both 393dp (Motorola) and 480dp (S24 Ultra) after every sprint.**
6. **Build APK with `--no-tree-shake-icons` flag** for every release build.

## SPRINT 1 — Critical Dashboard Fix
**Target date: Day 1**
**Files: 2**
**Impact: Fixes the most visible Motorola problem immediately**

### Problem
On 393dp Motorola: `_buildHeaderSection` has no height cap → takes ~140–165dp.
The `CardDeckView` gets only ~560dp. Dashboard widget cards shrink-wrap their content
(double-nested `SingleChildScrollView`) leaving large empty whitespace at the bottom.

### Changes

#### 1A — `lib/main.dart` · method `_buildDashboardGrid`

Wrap `_buildHeaderSection` with a proportional height cap:

```dart
// BEFORE:
if (!isLandscape) _buildHeaderSection(context),

// AFTER:
if (!isLandscape)
  ConstrainedBox(
    constraints: BoxConstraints(
      maxHeight: (MediaQuery.of(context).size.height * 0.22).clamp(100.0, 160.0),
    ),
    child: _buildHeaderSection(context),
  ),
```

**Why 22%:** On 780dp (Motorola) = 171dp max. On 852dp (iPhone 15) = 187dp max. On
932dp (Pro Max) = 205dp max → clamp to 160dp. On 667dp (iPhone SE) = 147dp max.
This gives the CardDeck at least 30dp more vertical space on Motorola.

#### 1B — `lib/ui/dashboard/widgets/net_worth_widget.dart` · method `buildContent()`

Find and remove the wrapping `SingleChildScrollView`. The outer
`_buildDashboardWidgetCard` in `main.dart` already provides scrollability via its own
`LayoutBuilder + ConstrainedBox(minHeight: constraints.maxHeight)` wrapper.

```dart
// BEFORE (approx line 792):
return SingleChildScrollView(
  physics: const BouncingScrollPhysics(),
  child: _NetWorthBody(
    accounts: visibleAccounts,
    ...
  ),
);

// AFTER — remove the SingleChildScrollView entirely:
return _NetWorthBody(
  accounts: visibleAccounts,
  totalSavings: totalSavings,
  totalInvestments: totalInvestments,
  totalCreditUsed: totalCreditUsed,
  totalNetWorth: totalNetWorth,
  momTrendPct: momTrendPct,
  isDark: isDark,
  healthData: healthData,
);
```

**Why:** `_NetWorthBody` uses `mainAxisSize: MainAxisSize.min` — correct. The outer
`ConstrainedBox(minHeight: constraints.maxHeight)` pads the remaining space. The inner
scroll was causing the `minHeight` to be applied to a scroll view that only sized to
content, not to the body directly.

### Verification
Build and test: on Motorola-width screen, Scorecard and Spending Insights cards should
now fill the full card area with no empty whitespace at the bottom.

## SPRINT 2 — Responsive Foundation Layer (New File)
**Target date: Day 2**
**Files: 1 new file**
**Impact: Enables all subsequent sprints — the entire framework**

### Create `lib/ui/styles/responsive_utils.dart`

This file must be created fresh. It provides:
- `Breakpoints` — breakpoint detection
- `RS` — responsive spacing (scales `Spacing.*` tokens)
- `RT` — responsive typography (scales `TypeScale.*` values)
- `RLayout` — responsive layout helpers

`ResponsiveTypography` already exists in `typography.dart` with `scaledFontSize()` and
`responsive()` — but uses wrong breakpoint ranges (375/414/768 instead of our device
matrix). Do NOT use it. The `RS`/`RT` below supersede it.

```dart
// lib/ui/styles/responsive_utils.dart

import 'package:flutter/material.dart';
import 'package:vittara_fin_os/ui/styles/design_tokens.dart';

// ─────────────────────────────────────────────────────────────────────────────
// Breakpoint detection
// ─────────────────────────────────────────────────────────────────────────────

enum AppBreakpoint { xs, sm, md, lg, xl, foldInner, tablet, tabletLarge }

class Breakpoints {
  Breakpoints._();

  static AppBreakpoint of(BuildContext context) {
    final size = MediaQuery.of(context).size;
    final w = size.width;
    final ratio = w / size.height;
    if (w >= 720 && w < 900 && ratio < 1.4) return AppBreakpoint.foldInner;
    if (w >= 1024) return AppBreakpoint.tabletLarge;
    if (w >= 768)  return AppBreakpoint.tablet;
    if (w >= 480)  return AppBreakpoint.xl;
    if (w >= 412)  return AppBreakpoint.lg;
    if (w >= 393)  return AppBreakpoint.md;
    if (w >= 360)  return AppBreakpoint.sm;
    return AppBreakpoint.xs;
  }

  static bool isTabletOrLarger(BuildContext context) {
    final bp = of(context);
    return bp == AppBreakpoint.tablet ||
        bp == AppBreakpoint.tabletLarge ||
        bp == AppBreakpoint.foldInner;
  }

  static bool isFoldInner(BuildContext context) =>
      of(context) == AppBreakpoint.foldInner;

  static bool isNarrowPhone(BuildContext context) {
    final bp = of(context);
    return bp == AppBreakpoint.xs || bp == AppBreakpoint.sm;
  }

  static bool isWidePhone(BuildContext context) {
    final bp = of(context);
    return bp == AppBreakpoint.lg || bp == AppBreakpoint.xl;
  }
}

// ─────────────────────────────────────────────────────────────────────────────
// RS — Responsive Spacing
// Scale factor: screenWidth / 480dp (reference). Clamped 0.78 – 1.15.
// Usage: RS.lg(context) instead of Spacing.lg
// ─────────────────────────────────────────────────────────────────────────────

class RS {
  RS._();

  static double factor(BuildContext context) =>
      (MediaQuery.of(context).size.width / 480.0).clamp(0.78, 1.15);

  static double xxs(BuildContext context) => Spacing.xxs * factor(context);
  static double xs(BuildContext context)  => Spacing.xs  * factor(context);
  static double sm(BuildContext context)  => Spacing.sm  * factor(context);
  static double md(BuildContext context)  => Spacing.md  * factor(context);
  static double lg(BuildContext context)  => Spacing.lg  * factor(context);
  static double xl(BuildContext context)  => Spacing.xl  * factor(context);
  static double xxl(BuildContext context) => Spacing.xxl * factor(context);
  static double xxxl(BuildContext context)=> Spacing.xxxl* factor(context);
  static double huge(BuildContext context)=> Spacing.huge* factor(context);

  static EdgeInsets symmetric(BuildContext context,
      {double h = 0, double v = 0}) =>
      EdgeInsets.symmetric(
        horizontal: h * factor(context),
        vertical: v * factor(context),
      );

  static EdgeInsets all(BuildContext context, double val) =>
      EdgeInsets.all(val * factor(context));

  static EdgeInsets screenH(BuildContext context) =>
      EdgeInsets.symmetric(horizontal: lg(context));
}

// ─────────────────────────────────────────────────────────────────────────────
// RT — Responsive Typography
// Per-breakpoint scale factors. Captions clamped to readable minimums.
// Usage: RT.title2(context) instead of TypeScale.title2
// ─────────────────────────────────────────────────────────────────────────────

class RT {
  RT._();

  static double factor(BuildContext context) {
    final w = MediaQuery.of(context).size.width;
    if (w < 360) return 0.86;  // xs: budget phones
    if (w < 393) return 0.91;  // sm: Motorola — THE critical range
    if (w < 412) return 0.96;  // md: iPhone 14/15
    if (w < 480) return 1.00;  // lg: Galaxy S22/S23
    if (w >= 768) return 1.10; // tablet
    return 1.04;               // xl: S24 Ultra reference
  }

  // Display / Hero
  static double display(BuildContext context) =>
      TypeScale.display * factor(context);
  static double hero(BuildContext context) =>
      TypeScale.hero * factor(context);

  // Titles
  static double largeTitle(BuildContext context) =>
      TypeScale.largeTitle * factor(context);
  static double title1(BuildContext context) =>
      TypeScale.title1 * factor(context);
  static double title2(BuildContext context) =>
      TypeScale.title2 * factor(context);
  static double title3(BuildContext context) =>
      TypeScale.title3 * factor(context);

  // Body
  static double headline(BuildContext context) =>
      TypeScale.headline * factor(context);
  static double body(BuildContext context) =>
      TypeScale.body * factor(context);
  static double callout(BuildContext context) =>
      TypeScale.callout * factor(context);
  static double subhead(BuildContext context) =>
      TypeScale.subhead * factor(context);

  // Small — clamped to prevent unreadably tiny text
  static double footnote(BuildContext context) =>
      (TypeScale.footnote * factor(context)).clamp(11.0, 14.0);
  static double caption(BuildContext context) =>
      (TypeScale.caption * factor(context)).clamp(10.0, 13.0);
  static double caption2(BuildContext context) =>
      (TypeScale.caption2 * factor(context)).clamp(9.0, 11.0);
}

// ─────────────────────────────────────────────────────────────────────────────
// RLayout — Responsive Layout Helpers
// ─────────────────────────────────────────────────────────────────────────────

class RLayout {
  RLayout._();

  /// Max width for modal/sheet content on tablet/fold inner screens
  static double sheetMaxWidth(BuildContext context) =>
      Breakpoints.isTabletOrLarger(context) ? 560.0 : double.infinity;

  /// Constrain widget to tablet-safe max width
  static Widget tabletConstrain(BuildContext context, Widget child) {
    if (!Breakpoints.isTabletOrLarger(context)) return child;
    return Center(
      child: ConstrainedBox(
        constraints: const BoxConstraints(maxWidth: 560),
        child: child,
      ),
    );
  }

  /// Dynamic childAspectRatio from LayoutBuilder width
  /// Usage: in GridView.builder delegate, use this instead of hardcoded ratio
  static double gridAspectRatio(
    BoxConstraints constraints,
    int crossAxisCount, {
    double crossAxisSpacing = 12.0,
    double targetRatio = 1.1,
  }) {
    final itemWidth =
        (constraints.maxWidth - (crossAxisCount - 1) * crossAxisSpacing) /
            crossAxisCount;
    return itemWidth / (itemWidth * targetRatio);
  }

  /// Whether to show 2-column split layout
  static bool shouldSplit(BuildContext context) =>
      AppStyles.isLandscape(context) || Breakpoints.isTabletOrLarger(context);
}
```

**Note:** Import this file wherever `RS.*`, `RT.*`, `Breakpoints.*`, `RLayout.*` are used.
Add the import: `import 'package:vittara_fin_os/ui/styles/responsive_utils.dart';`

## SPRINT 4 — All GridView `childAspectRatio` Fixes
**Target date: Day 4**
**Files: 13** (all files with `childAspectRatio`)
**Impact: Prevents cell overflow/clipping on narrow phones**

### Problem
Fixed `childAspectRatio: 0.78` or `1.2` was computed for ~480dp cards. On 393dp the
same ratio produces cells that are either too tall (overflow) or too wide (wrong
proportion).

### Fix Pattern (apply to ALL 13 files below)

```dart
// BEFORE — anywhere you see:
GridView.builder(
  gridDelegate: SliverGridDelegateWithFixedCrossAxisCount(
    crossAxisCount: 2,
    childAspectRatio: 1.2,     // ← hardcoded
    crossAxisSpacing: 12,
    mainAxisSpacing: 12,
  ),
  ...
)

// AFTER — wrap entire GridView in LayoutBuilder:
LayoutBuilder(
  builder: (context, constraints) {
    final cols = 2;
    final spacing = 12.0;
    final itemW = (constraints.maxWidth - (cols - 1) * spacing) / cols;
    return GridView.builder(
      gridDelegate: SliverGridDelegateWithFixedCrossAxisCount(
        crossAxisCount: cols,
        childAspectRatio: itemW / (itemW * 1.1),  // preserve original ratio
        crossAxisSpacing: spacing,
        mainAxisSpacing: spacing,
      ),
      ...
    );
  },
)
```

**Adjust `1.1` to match the original target ratio for each file.**

## SPRINT 6 — Critical Fixed Heights Audit & Fix
**Target date: Day 7–8**
**Files: ~35 (structural height values > 40dp)**
**Impact: Prevents layout overflow and wasted space on small/large screens**

### Problem
193 structural `height: N` values above 40dp exist in the codebase. Not all need
changing — spacing-only `SizedBox` gaps are fine. The risky ones are containers,

```dart
// AFTER:
Container(
  constraints: const BoxConstraints(minHeight: 110, maxHeight: 160),
  child: optionCard,

```

#### `lib/ui/net_worth_page.dart` — any fixed chart heights
Replace `SizedBox(height: 140)` / `SizedBox(height: 200)` chart containers with
`AspectRatio(aspectRatio: 16/9)` or `LayoutBuilder`-computed heights.

#### `lib/ui/spending_insights_screen.dart` — chart section heights
Replace fixed `height:` values on chart containers with `AspectRatio(16/9)`.

#### `lib/ui/manage/tax_summary_screen.dart` — summary card heights
Replace fixed heights with `IntrinsicHeight` or content-driven sizing.

#### `lib/ui/dashboard/dashboard_settings_modal.dart` — modal interior
Replace fixed SizedBox heights with `RS.xl(context)` spacers.

### Rule for this sprint
For every `height: N` where N > 40dp, ask:
- Is this a **gap/spacer** SizedBox between widgets? → Leave it OR convert to `RS.lg(context)`
- Is this a **structural container** (card, list item, image)? → Convert to `ConstrainedBox`
  or `AspectRatio` or `Flexible`
- Is it in a `CustomPaint`/chart? → Use `LayoutBuilder` or `AspectRatio(16/9)`

### 14A — Create `lib/ui/widgets/adaptive_split_view.dart`

Extend `LandscapeSplitView` (which handles landscape phones) to also handle
tablet portrait and fold inner screens:

```dart
class AdaptiveSplitView extends StatelessWidget {
  final Widget leading;    // Left pane (nav/list)
  final Widget trailing;   // Right pane (detail/content)
  final double leadingFraction; // Default 0.38

  const AdaptiveSplitView({
    super.key,
    required this.leading,
    required this.trailing,
    this.leadingFraction = 0.38,
  });

  bool _shouldSplit(BuildContext context) =>
      AppStyles.isLandscape(context) || Breakpoints.isTabletOrLarger(context);

  @override
  Widget build(BuildContext context) {
    if (!_shouldSplit(context)) return trailing;
    final w = MediaQuery.of(context).size.width;
    final leadW = w * leadingFraction;
    return Row(
      children: [
        SizedBox(width: leadW, child: leading),
        VerticalDivider(width: 1, color: AppStyles.getDividerColor(context)),
        Expanded(child: trailing),
      ],
    );
  }
}
```

## Global Red Flags & Rules

### Red Flag 1 — `NetWorthWidget` bypasses `BaseDashboardWidget.build()`
`NetWorthWidget.build()` calls `buildContent()` directly, bypassing the
card decoration in `BaseDashboardWidget`. Fixes to `BaseDashboardWidget`
(Sprint 3A) do NOT automatically apply to `NetWorthWidget`.
**Rule:** After every dashboard widget fix in Sprint 3, verify `NetWorthWidget`
is not affected and fix separately.

### Red Flag 2 — `const EdgeInsets` in `Spacing.*`
`Spacing.screenPadding` and `Spacing.cardPadding` are `const EdgeInsets` — used
at 100+ sites as `const Spacing.screenPadding`. Making them dynamic would require
removing `const` everywhere.
**Rule:** Never change `Spacing.*` constants. Use `RS.*` at new/modified call sites only.

### Red Flag 3 — `heroNumberStyle` default `fontSize: 40`
Long rupee numbers (₹12,34,567) at `fontSize: 40` WILL overflow on 393dp.
**Rule:** Wherever `heroNumberStyle(context)` is called with the default fontSize,
wrap the `Text` widget in `FittedBox(fit: BoxFit.scaleDown)`.

### Red Flag 4 — Double `SingleChildScrollView` in Dashboard Widgets
This is the root cause of the Motorola empty-space bug. After Sprint 1 and Sprint 3,
verify there are zero double-nested scrollers in any dashboard widget.
Run: `grep -rn "SingleChildScrollView" lib/ui/dashboard/` after Sprint 3.

### Red Flag 5 — `childAspectRatio` with screen-width assumptions
After Sprint 4, run `grep -rn "childAspectRatio" lib/` to verify all 16 instances
are now `LayoutBuilder`-computed. Any remaining hardcoded ratios will break on
360dp fold outer screens.

### Red Flag 6 — `SubtleParticleOverlay` on tablet
`particleCount: 42` on a 768dp+ screen causes visible rendering lag. Reduce in
Sprint 13 for all occurrences.

### Red Flag 7 — `AppStyles.titleStyle` uses `fontSize: 18` (not in TypeScale)
This value is between `TypeScale.title3 (17)` and `TypeScale.title2 (20)`.
Do not change `titleStyle` itself — instead when migrating call sites in Sprint 7,
use `RT.title2(context)` at the specific widget, not via `titleStyle`.
